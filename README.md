# LLM-Based RTL Evaluation Framework

A stage-based evaluation pipeline for assessing Verilog RTL generated by Large Language Models (LLMs).

## Validation Stages

| Stage | Name | What it checks | Tool |
|-------|------|----------------|------|
| P | Port Signature | Module header, port declarations, I/O direction | Python + Yosys |
| C | Compilation | Syntax errors, Verilog legality | Icarus Verilog |
| E | Elaboration | Module hierarchy, parameters | Yosys |
| M | Module Completeness | No stubs, no placeholders, complete logic | Python |
| F | Functional Equivalence | Matches reference implementation | Yosys |
| S | Synthesis | Gate-level netlist generation | Cadence Genus |

## Design Naming Convention

The framework uses a single `DESIGN` variable to control all file paths. When you set:

```python
DESIGN = "half_adder"
All paths are automatically derived:

Path Variable	Actual Path
PROMPT_FILE	prompts/half_adder.txt
RTL_FILE	rtl/half_adder.v
RAW_FILE	rtl/raw_output.txt
REFERENCE_PATH	reference/half_adder/ (or reference/half_adder.v)
RESULT_DIR	results/half_adder/
This ensures consistency across all files for a design and enables batch testing of multiple designs.

Creating a Design
Step 1: Create Prompt
File: prompts/half_adder.txt

text
Design a half adder module.

Module name: half_adder
Inputs: a, b (1-bit)
Outputs: sum, cout (1-bit)

Logic:
- sum = a XOR b
- cout = a AND b

Use assign statements. Provide only Verilog code.
Step 2: Create Reference
Single file: reference/half_adder.v

verilog
module half_adder(a, b, sum, cout);
  input a, b;
  output sum, cout;
  assign sum = a ^ b;
  assign cout = a & b;
endmodule
Multiple files: reference/cpu/ directory

text
reference/cpu/
├── cpu_top.v
├── alu.v
└── control.v
If the reference top module has a different name, set REFERENCE_TOP_NAME in main.py.

Step 3: Add to Test Suite
python
# In run_all.py
DESIGNS = ["half_adder", "cpu", "your_design"]

Metrics
Stage-wise Pass Rates
Metric	Formula	Description
PPS	(1/N)·∑p_i	Port Signature pass rate
CR	(1/N)·∑c_i	Compile pass rate
ER	(1/N)·∑e_i	Elaboration pass rate
MC	(1/N)·∑m_i	Module completeness rate
FE	(1/N)·∑q_i	Functional equivalence rate
Conditional Yields
Metric	Formula	Interpretation
CR|PPS	∑c_i / max(1,∑p_i)	Compile given port signature
ER|CR	∑e_i / max(1,∑c_i)	Elaboration given compile
MC|ER	∑m_i / max(1,∑e_i)	Complete given elaboration
FE|MC	∑q_i / max(1,∑m_i)	Functional given complete
Success Metrics
Metric	Formula	Description
E2E@K	(1/N)·#{i: A_i ≤ K}	Success within K attempts
SEY@K	(1/N)·#{i: max(c_i·e_i)=1}	Synthesis-eligible yield
ETS	(1/	S	)·∑(A_i-1)	Avg refinements before success
TTFP	(1/	S	)·∑t_i	Avg time to first success
Failure Analysis
Metric	Values
First-failure stage	P, C, E, M, F, or None
Root cause	compiler, not-elaborated, partial_module, functional_mismatch
Interpreting Results
Observation	Meaning
Low PPS	LLM struggles with Verilog syntax
Low CR|PPS	Port declaration issues
Low ER|CR	Hierarchy/parameter problems
Low MC|ER	Incomplete module generation
Low FE|MC	Logic doesn't match specification
High ETS	LLM needs many refinements
Low SEY@K	Most designs cannot reach synthesis
Target Values
Metric	Good Value
E2E@3	>50%
SEY@3	>70%
ETS	<2
TTFP	<60 seconds
Troubleshooting
Problem	Solution
Ollama connection refused	ollama serve &
HuggingFace API error	Check HF_API_KEY and model name
SSH permission denied	Check password or use SSH keys
JSON parse error	Ensure __JSON_START__ and __JSON_END__ markers in output
Yosys equivalence fails	Set REFERENCE_TOP_NAME correctly
No reference file	Create reference/{design}.v or reference/{design}/ directory
