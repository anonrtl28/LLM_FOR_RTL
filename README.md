# LLM-Based RTL Evaluation Framework

This repository provides a stage-based evaluation pipeline for assessing Verilog RTL generated by Large Language Models (LLMs).

The framework evaluates generated designs across multiple stages including:
- Port Signature Check (PPS)
- Compilation (CR)
- Elaboration (ER)
- Module Completeness (MC)
- Functional Equivalence (FE)
- Synthesis (S)

---

## Required Directory Structure

Before running the framework, users must create the following directory structure:
project/
├── main.py
├── run_all.py
├── metrics.py
├── prompts/
│ ├── <design>.txt
├── reference/
│ ├── <design>.v
├── rtl/
├── results/


---

##  Input Requirements

For each design `<design>`:

- `prompts/<design>.txt`  
  → Natural language prompt describing the RTL module

- `reference/<design>.v`  
  → Golden/reference RTL for functional equivalence checking

---

## How to Run

### Step 1: Run evaluation (K attempts per design)
ensure that the parameters such as SSH keys, LLM model name, design name etc, are defined in main.py as per the requirement
python3 run_all.py
This generates:
results.csv

---

### Step 2: Compute metrics

python3 metrics.py

---

## Metrics Reported

The framework computes:

- Stage-wise pass rates (PPS, CR, ER, MC, FE)
- Conditional yields
- End-to-End Success @ K (E2E@K)
- Synthesis Eligibility Yield @ K (SEY@K)
- Expected Trials to Success (ETS)
- Time to First Pass (TTFP)
- First Failure Stage distribution

---

## Notes

- Ensure required tools are installed:
  - `yosys`
  - `iverilog`
- API keys must be configured inside `main.py`
- The framework assumes a maximum of K=3 attempts (modifiable in `run_all.py`)
